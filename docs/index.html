Subtitle Format Conversion for FFmpeg

FFmpeg can transcode between virtually any audio and video format. Subtitles are the exception. Converting between text (SRT, ASS, WebVTT) and bitmap (PGS, DVB, DVD, XSUB) formats has been broken for over a decade.

"Subtitle encoding currently only possible from text to text or bitmap to bitmap"

This is FFmpeg bug #3819. This repository contains 21 patches that fix it — in both directions. 64 bitmap encoders (16 text decoders × 4 bitmap encoders) and 24 text encoders (4 bitmap decoders × 6 text encoders).

Approach

Prior art. Two attempts stalled. Coza (2022) put libass in libavcodec — a dependency violation. softworkz (2021–2025) proposed full subtitle filter infrastructure across 25 patches and 89 files; design questions remain unresolved.

This series takes the minimal path. libass (rendering) and Tesseract (OCR) already live in libavfilter. Small utility APIs there, called from fftools in the same pattern as sub2video, handle both directions. No filter infrastructure. No AVFrame changes.

Patches

Branch: pgs-series. Series A/B/C are independent and reviewable in parallel. D depends on all three. E and F are independent additions.

A PGS Encoder · 1 patch
  • Full composition state machine with automatic Display Set type selection
  • palette_update_flag for palette-only updates (no bitmap retransmission)
B Quantization API · 11 patches
  • NeuQuant, Median Cut, and ELBG quantizers via av_quantize_* API
  • OkLab perceptual colour space, palette remapping and dithering from vf_paletteuse
  • Region-weighted palette generation for overlapping subtitle events
  • Algorithm comparison with PSNR and timing data
C Subtitle Renderer · 2 patches
  • Public avfilter_subtitle_render_* API wrapping libass
  • Multi-timepoint rendering for animation; stub when libass is unset
D fftools Orchestration · 4 patches
  • Text-to-bitmap pipeline: render, quantize, encode
  • Rect splitting, animation detection, event coalescing
E GIF Encoder · 1 patch
  • Direct RGBA-to-GIF encoding via av_quantize_* and ff_palette_map_apply
  • Configurable quantizer (Median Cut default) and dithering (Floyd-Steinberg default)
F OCR (Bitmap to Text) · 2 patches
  • Tesseract OCR integration via avfilter_subtitle_ocr_* API
  • Bitmap deduplication for PGS fade sequences; PSM fallback for RTL scripts
  • Grayscale conversion (binary extraction for DVD, luminance-alpha for PGS)

OCR (Bitmap to Text)

The reverse direction: PGS, DVB, DVD, and XSUB bitmap subtitles to SRT, ASS, WebVTT, or MOV text via Tesseract OCR. Tested across 114 languages with UDHR Article 1 roundtrip (text → PGS → text). 105 pass, 9 fail (Tesseract training data limitations).

# PGS to SRT (bitmap dedup on by default)
ffmpeg -i movie.mkv -map 0:s:0 -c:s srt output.srt

# Japanese DVD subtitles
ffmpeg -i movie.mkv -map 0:s:0 -c:s srt -sub_ocr_lang jpn output.srt

# PGS to ASS with position mapping
ffmpeg -i movie.mkv -map 0:s:0 -c:s ass output.ass

Bitmap deduplication skips OCR on palette-only changes (PGS fade sequences). PSM fallback handles RTL and complex scripts. Per-language preserve_interword_spaces tuning prevents word merging in Arabic, Hebrew, and Persian.

The -eng release bundle includes English tessdata. Additional languages available from the tessdata release.

Spec Compliance

Decoder model grounded in the Panasonic and Sony HDMV patents (US20090185789A1, US8638861B2, US7620297B2). Buffer sizes, transfer rates, and palette limits taken from the patents, not reverse-engineered from player behaviour.

SUPer by cubicibo used as a hardware-validated reference.

Each target is available in two variants: a minimal bundle (ffmpeg + ffprobe) and an -eng bundle that includes English OCR training data. For other languages, download .traineddata files from the tessdata release (124 languages) and place them in a tessdata/ directory next to the binary.

Static dependency chain: zlib 1.3.1, libpng 1.6.55, freetype 2.13.3, fribidi 1.0.16, graphite2 1.3.14, ICU 78.2, harfbuzz 10.2.0, libass 0.17.4, leptonica 1.87.0, tesseract 5.5.2. All linked statically. Only platform-mandated dynamic libraries remain (glibc on Linux, libSystem on macOS, kernel DLLs on Windows).

Usage

# Text to PGS (Blu-ray bitmap subtitles)
ffmpeg -i input.mkv -c:s pgssub -c:v copy -c:a copy output.mkv

# Text to PGS (.sup file for muxing tools)
ffmpeg -i subtitles.srt -s 1920x1080 output.sup

# PGS/DVD/DVB bitmap to SRT text (OCR)
ffmpeg -i input.mkv -c:s srt output.mkv

# PGS to ASS with position mapping
ffmpeg -i input.mkv -c:s ass output.mkv

# Japanese DVD subtitles to SRT
ffmpeg -i input.mkv -c:s srt -sub_ocr_lang jpn output.srt

# Remux subtitles without conversion
ffmpeg -i input.mkv -c:s copy -c:v copy -c:a copy output.mp4

# Choose quantization method for PGS encoding
ffmpeg -i input.mkv -c:s pgssub -quantize_method mediancut output.mkv
