FFmpeg Subtitle Tools

Stable: FFmpeg 8.0.1 Pre-built FFmpeg binaries with full subtitle format conversion. Text to bitmap, bitmap to text, and everything between.

FFmpeg can transcode between virtually any audio and video format. Subtitles are the exception — converting between text (SRT, ASS, WebVTT) and bitmap (PGS, DVB, DVD) formats has been broken for over a decade (#3819, #6843). These builds fix that.

What's included

PGS Encoder

Full Blu-ray PGS subtitle encoder with composition state machine, palette animation, and decoder model compliance per HDMV patent specifications.

Text to Bitmap

Automatic conversion from any text subtitle (SRT, ASS, WebVTT, etc.) to any bitmap format. 64 new encoder pairs via libass rendering and NeuQuant quantization.

Bitmap to Text (OCR)

Tesseract OCR integration for converting PGS, DVB, and DVD bitmap subtitles to SRT, ASS, or WebVTT. 24 new encoder pairs. 114 languages tested.

GIF Encoder

Direct RGBA-to-GIF encoding without the palettegen+paletteuse filter chain. Configurable quantizer and dithering.

Download

Each target is available in two variants: a minimal bundle (ffmpeg + ffprobe) and an -eng bundle that includes English OCR training data. For other languages, download .traineddata files from the tessdata release (124 languages) and place them in a tessdata/ directory next to the binary.

Static dependency chain: zlib 1.3.1, libpng 1.6.55, freetype 2.13.3, fribidi 1.0.16, graphite2 1.3.14, ICU 78.2, harfbuzz 10.2.0, libass 0.17.4, leptonica 1.87.0, tesseract 5.5.2. All linked statically. Only platform-mandated dynamic libraries remain (glibc on Linux, libSystem on macOS, kernel DLLs on Windows).

Usage

# Text to PGS (Blu-ray bitmap subtitles)
ffmpeg -i input.mkv -c:s pgssub -c:v copy -c:a copy output.mkv

# Text to PGS (.sup file for muxing tools)
ffmpeg -i subtitles.srt -s 1920x1080 output.sup

# PGS/DVD/DVB bitmap to SRT text (OCR)
ffmpeg -i input.mkv -c:s srt output.mkv

# Japanese DVD subtitles to SRT
ffmpeg -i input.mkv -c:s srt -sub_ocr_lang jpn output.srt

# Direct RGBA to GIF (no filter pipeline)
ffmpeg -i input.mp4 -c:v gif out.gif

# Remux subtitles without conversion
ffmpeg -i input.mkv -c:s copy -c:v copy -c:a copy output.mp4

About this release

This is the first stable release based on an official FFmpeg release branch. Previous builds tracked FFmpeg master, which changes daily and can introduce unrelated regressions.

The n8.0.1-pgs.0 tag applies 21 patches to FFmpeg's 8.0.1 release (the latest point release of the 8.0 series). The base code is unchanged — only our subtitle patches are added on top. All upstream FATE tests pass.

GIF RGBA pipeline fix. This release also fixes a bug in the new GIF RGBA quantization path where ff_palette_map_init internally sorts palette entries for KD-tree construction, but the returned indices were being written against the original unsorted palette. The result was all-white or garbled GIF output when encoding directly from RGBA. The fix uses the mapper's sorted palette for the output frame.

The patch series continues to be maintained against FFmpeg master for eventual upstream submission. See the development page for patch-level detail, upstream strategy, and the full series breakdown.
