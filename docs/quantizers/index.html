Color Quantization: Three Algorithms, One API

Phase 5 of the FFmpeg subtitle patch series integrates three color quantization algorithms into a single av_quantize_* API. This page presents hard data comparing their quality and performance on representative subtitle images.

All measurements from a 128×64 test image with quality=10. PSNR computed per-pixel in sRGB against the original, averaged over RGB channels.

The Algorithms

ELBG

Patane & Russo, 2001

Enhanced LBG vector quantizer. Iteratively refines codebook entries by splitting high-distortion cells and merging low-utility ones. General N-dimensional; wrapped here for 4D RGBA.

Median Cut

Heckbert, 1982

Recursively splits colour space along the axis of greatest variance. Deterministic and predictable. Extracted from FFmpeg's vf_palettegen filter. Uses OkLab perceptual space internally.

NeuQuant

Dekker, 1994

Kohonen self-organising map in colour space. Neurons compete to represent input colours via competitive learning. Fast, but non-deterministic and quality-sensitive to learning rate.

The Story

This patch series started with a simple goal: give FFmpeg a PGS subtitle encoder so you can convert text subtitles to Blu-ray format. That encoder needed color quantization — reducing rendered RGBA images to 256-color palettes.

Phase 2 introduced NeuQuant as the first quantizer, wrapped in a public av_quantize_* API in libavutil. It worked well, but NeuQuant is a neural network — non-deterministic, hard to reason about, and not always the best choice.

Phase 5 completes the quantization story by integrating two additional algorithms:

  1. Median Cut extraction — The algorithm from vf_palettegen (FFmpeg's GIF palette generator) moved to libavutil. Core histogram + box-splitting code extracted; the filter retains its AVFilter boilerplate. Incremental API (ff_mediancut_add_color) bridges the gap between the filter's frame-by-frame accumulation and the library's flat-buffer interface.
  2. vf_palettegen refactor — The filter shrinks from 586 to 324 lines. All algorithm code replaced with ff_mediancut_* calls. Bit-for-bit identical output on all three stats modes (full/diff/single).
  3. ELBG move — ELBG was in libavcodec with avpriv_ prefix despite having zero libavcodec dependencies. Moved to libavutil where it belongs. Five consumers updated (cinepakenc, a64multienc, msvideo1enc, roqvideoenc, vf_elbg).
  4. ELBG quantizer — General N-dimensional vector quantizer wrapped for 4D RGBA. Pixels become int arrays, ELBG finds codebook entries, entries become palette colors. Quality parameter maps to iteration steps.
  5. quantize_method option — New -quantize_method option on the PGS encoder. fftools reads it from the encoder's private data and passes it through to all quantize call sites. Three choices: elbg, mediancut, neuquant (default).

Usage

# Default (NeuQuant) — fast, good quality
ffmpeg -i input.srt -c:s pgssub -s 1920x1080 output.sup

# Median Cut — deterministic, excellent on structured images
ffmpeg -i input.ass -c:s pgssub -quantize_method mediancut -s 1920x1080 output.sup

# ELBG — highest quality on complex gradients, slower
ffmpeg -i input.ass -c:s pgssub -quantize_method elbg -s 1920x1080 output.sup

Recommendation

For most subtitle work: use the default (NeuQuant). Subtitles typically have fewer than 100 unique colors. All three algorithms achieve perfect or near-perfect PSNR at this complexity. NeuQuant is the fastest and produces excellent results.

For complex ASS with gradients or many colors: consider Median Cut or ELBG. The data shows 10–15 dB improvement on gradient stress tests. ELBG is best on pure gradients; Median Cut is best on structured multi-region images and runs faster.

Methodology

  • Test images: 128×64 procedurally generated (no external dependencies)
  • Five scenarios: white text on black, 4-quadrant multicolour, karaoke highlight, RGB gradient, full HSV sweep
  • Three palette sizes: 16, 64, 256 (matching DVD, DVB, and PGS maximums)
  • Quality parameter: 10 (default) for all algorithms
  • PSNR: per-pixel squared error in sRGB, averaged over RGB channels, 10×log10(255²/MSE)
  • Timing: clock_gettime(CLOCK_MONOTONIC), includes both palette generation and pixel mapping
  • Platform: Linux x86_64, GCC, single-threaded
